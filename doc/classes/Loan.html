Loan Class

Defined in: server/public/classes/Loan.js:10

Loan is meant to be shared between server and client side code It's properties match those in the GraphQL scheme (insofar as possible) It's constructor takes no arguments. All properties must be set with the results of a GraphQL API call/MongoDB call. However: Note that contained objects, like 'geocode' or 'image' that have their own types in the GraphQL API do not have their own types here. They are simply treated as vanilla JSON objects
